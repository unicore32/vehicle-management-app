# 仕様書

このディレクトリには、Markdown ベースの機能仕様を配置する。

## 推奨フォーマット
- 1 つの機能または変更につき 1 ファイルにする
- 短くして、実装判断に使える内容へ絞る
- 長い説明文ではなく、現時点の決定事項を書く
- 変更後は `pnpm run lint:docs` を通す

## テンプレート

# <機能名>

## 目的
- この機能が解決する課題

## スコープ
- 含めるもの
- 対象外とするもの

## 挙動
- 主要なユーザーフロー
- エッジケース
- エラーハンドリング

## データモデル
- テーブル / フィールド / リレーション

## 画面/UI
- 画面
- アクション
- empty / loading / error 状態

## メモ
- 未解決の論点
- 将来のフォローアップ
