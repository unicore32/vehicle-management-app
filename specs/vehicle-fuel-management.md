# 車両管理 / 燃費管理（先行仕様）

## 目的
- GPS セッションを車両単位で管理できるようにし、将来の燃費管理と整備記録の土台を作る。
- 記録開始前に車両とメーター距離を任意入力できるようにし、記録終了時とセッション詳細からも補完・修正できるようにする。
- GPS 距離と車両メーター距離を分けて扱い、後続の給油記録ではメーター距離ベースの集計へつなげる。

## 関連仕様
- GPS セッションの共通データモデルと画面構成は [specs/gps-session-foundation.md](specs/gps-session-foundation.md) を参照する。
- GPS セッションの記録ライフサイクルは [specs/gps-session-management.md](specs/gps-session-management.md) を参照する。

## スコープ
### 今回の対象
- 車両マスタの最小管理
- GPS 記録開始前の車両選択
- GPS 記録開始時の開始メーター距離入力
- GPS 記録終了時の終了メーター距離入力
- セッション詳細での車両情報と開始/終了メーター距離の照会・修正
- 将来の給油記録追加を前提にしたデータモデル整理

### 今回の対象外
- 給油記録の登録 UI と燃費計算
- 給油履歴一覧、満タン法判定、費用集計
- 整備記録の登録・通知・履歴管理
- OBD や外部サービスからの車両データ自動取得
- 複数ユーザー/クラウド同期

## 用語
- **車両**: GPS セッションや今後の給油記録が紐づく管理対象。
- **開始メーター距離**: セッション開始時点でユーザーが入力したオドメーター値。
- **終了メーター距離**: セッション終了時点でユーザーが入力したオドメーター値。
- **メーター差分**: `終了メーター距離 - 開始メーター距離`。GPS 距離とは別の値として扱う。

## 挙動
### 全体方針
- 車両選択は任意とし、未選択のまま GPS 記録を開始できる。
- メーター距離入力も任意とする。
- 車両が未選択のセッションでは、開始/終了メーター距離は入力しない。
- GPS の `distance_m` は従来どおり GPS セッションの集計値として保持し、車両メーター差分で上書きしない。
- 将来の燃費管理では、セッションに紐づいたメーター距離を給油記録と組み合わせる前提とする。

### 車両マスタ
- 車両管理画面を追加し、1 台以上の車両を登録できるようにする。
- 初回実装の車両登録項目は次の最小構成とする。
- `display_name`（必須）
- `is_active`（論理有効フラグ）
- 初回実装では `manufacturer` と `model_name` は持たない。
- 選択 UI は `display_name` のみを表示する。
- 車両は物理削除を前提にせず、初期版ではアーカイブ相当の `is_active = false` を使える構造にする。
- 進行中セッションに紐づく車両の編集やアーカイブは行えてよいが、進行中セッション自体の車両変更は今回対象外とする。

### 記録開始前フロー
- Home 画面で `記録を開始` を押した時点では即時開始せず、開始前シートまたはモーダルを開く。
- 開始前 UI には次を表示する。
- 車両選択
- 開始メーター距離入力（km、小数入力は許可しない）
- `車両未選択で開始` を含む開始アクション
- 車両一覧が空の場合でも記録開始は可能とし、`車両を追加` 導線を併設する。
- 車両未選択時は開始メーター距離入力を非表示または無効化する。
- 過去に同じ車両で終了メーター距離が保存されている場合、その最新値を開始メーター距離の候補として表示してよい。
- 候補値は自動確定せず、ユーザーが明示的に採用または編集できる状態にする。

### 記録終了フロー
- `停止・完了` を押した時点では即時終了せず、完了確認と終了メーター距離入力を兼ねた単一のシートまたはモーダルを開く。
- 完了 UI には最低限次を表示する。
- 紐づいている車両名または未選択表示
- 開始メーター距離（入力済みの場合）
- 終了メーター距離入力
- 保存して完了
- 車両未選択のセッションでは終了メーター距離入力は表示しない。
- 車両選択済みでも、終了メーター距離を空のまま完了できる。
- 開始メーター距離と終了メーター距離の両方がある場合、終了値は開始値以上でなければならない。
- 初回実装では、同一車両の過去セッションを参照した整合性チェックや警告は行わない。

### セッション詳細
- セッション詳細に `車両情報` セクションを追加する。
- 表示項目は次とする。
- 車両名
- 開始メーター距離
- 終了メーター距離
- メーター差分（両方がある場合のみ）
- GPS 距離
- セッション詳細から車両情報を編集できる。
- 編集対象は `vehicle_id`、開始メーター距離、終了メーター距離とする。
- 編集後はセッション詳細表示と一覧キャッシュを更新する。
- 車両未選択の既存セッションにも、後から車両を紐づけられるようにする。
- 後から車両を紐づけた場合、開始/終了メーター距離も同じ編集導線でまとめて設定できるようにする。

### セッション詳細編集ルール
- 完了済みセッションのみ車両情報を編集対象とする。
- `recording` または `paused` のアクティブセッションでは、車両とメーター距離の変更は行わない。
- 編集時に車両を変更しても GPS ポイントやセッション距離は変更しない。
- 編集時に開始/終了メーター距離を更新しても GPS サマリーは再計算しない。
- 将来の給油記録実装後は、車両変更やメーター変更が給油整合性へ影響するため、より強い制約を追加する余地を残す。

### 既存セッションの扱い
- 既存の `vehicle_id` 未設定セッションはそのまま表示できる必要がある。
- 既存の `開始メーター距離` / `終了メーター距離` 未設定セッションは `未入力` と表示する。
- 一括移行や過去セッションへの自動補完は今回対象外とする。

## データモデル
### vehicles
- `id`
- `display_name`
- `is_active`
- `created_at`
- `updated_at`

### sessions 追加項目
- `vehicle_id` nullable
- `odometer_start_km` nullable
- `odometer_end_km` nullable

### データモデル方針
- `vehicle_id` は `vehicles.id` を参照する。
- メーター距離は km の整数で保持する。
- GPS 距離は `sessions.distance_m` に残し、メーター距離とは別の列で管理する。
- 初回実装では車両名スナップショットを保持せず、表示時に `vehicle_id` から現在の車両名を参照する。
- 車両は物理削除せず `is_active` で管理するため、過去セッションの参照は維持できる前提とする。
- 将来の給油記録は `vehicle_id` と給油時メーター距離を必須にする想定で設計する。

### 将来拡張の見込み
- `fuel_logs`
- `id`
- `vehicle_id`
- `filled_at`
- `odometer_km`
- `fuel_volume_l`
- `unit_price`
- `total_price`
- `is_full_tank`
- `note`
- `created_at`
- `updated_at`

## 画面/UI
### Home
- `記録を開始` 押下で開始前シートを開く。
- シートで確定後にセッション作成と GPS 記録開始を行う。
- 車両選択済みで開始した場合、ボトムパネルまたはコントロール近辺に車両名を軽く表示してよい。

### セッション詳細
- サマリーカードとは別に `車両情報` セクションを置く。
- 編集アクションはインラインボタンまたは別シートで提供する。
- 表示優先度は `車両名 > メーター差分 > 開始/終了値` とする。

### 設定
- Settings から `車両管理` 画面へ遷移できるようにする。
- 車両管理画面では車両一覧、新規追加、編集、アーカイブを行えるようにする。

## バリデーション
- 車両未選択時にメーター距離は保存しない。
- メーター距離は 0 以上の整数のみ許可する。
- 終了メーター距離が開始メーター距離より小さい場合は保存不可とする。
- 数値変換できない入力は即時エラー表示する。
- 非常に大きい値の上限はアプリ内で固定せず、現実的でない値は将来の改善対象に留める。
- 初回実装では、同一車両の前回終了メーター距離との前後関係チェックは行わない。

## 初回実装の確定事項
- 車両登録項目は `display_name` と `is_active` のみとする。
- 記録終了時の UI は、確認と終了メーター距離入力を兼ねた 1 つのシートまたはモーダルとする。
- メーター距離の整合性チェックは同一セッション内の `開始 <= 終了` のみ行う。
- 同一車両の過去セッションを参照した警告や自動補正は入れない。
- `vehicle_display_name_snapshot` は初回実装では持たない。

## メモ
- 初回実装では、燃費そのものの計算よりも `車両とセッションを確実に結びつけること` を優先する。
- 実装コストを抑えるため、初回実装では車両名表示とメーター距離保存に必要な最小列だけを追加する。
- 給油管理を追加する段階では、セッション単位のメーター差分を直接集計に使うのではなく、給油イベント間のオドメーター差分を主軸に再整理する。
